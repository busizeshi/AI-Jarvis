# 个人知识库项目预期

打造一个个人版本的知识库平台，旨在帮助用户实现知识的存储，检索，复习，提醒，以及搭建自己工作流导出各种文件类型的文档。
之后可能会考虑文档的分享，社区等功能。主要针对用户：知识存储，快速学习，经验总结，效率提升，文档快速编辑，碎片化知识提取整合。

平台的核心差异点在于：**以 GraphRAG 构建的个人知识图谱为底座**，把「零散笔记」升级为「可推理、可关联、可复习」的结构化知识网络，并通过 Agent 工作流让知识主动为用户服务（提醒、计划、生成、导出）。

# 预期功能

## 知识存储

1. 知识库以及文档的组织存储
   - 支持创建自定义知识库，每个知识库可以包含多个文档。
   - 知识库的类型
     - 手机资源存储型：类似手机存储的云空间版本，可以存储文档，pdf，ppt，excel，markdown等文件类型。也可以存储图片，视频，音频等，不可编辑，可以支持云存储或者是手机存储
     - 零碎小计类型：用于存储一些碎片化的知识，例如会议记录，项目进度，个人学习记录以及解析网站内容等。可编辑
     - 个人笔记存储类型：存储个人的笔记，例如学习笔记，工作笔记，个人经验等。可编辑
     - 个人日记存储类型：存储个人的日记，可编辑
     - 记忆卡类型：用于存储一些个人的记忆卡片，比如知识点卡片，错题卡片等。可编辑
   - 笔记类型
     - 纯笔记： 类似语雀或者说是markdown格式的笔记，用户可以自由编辑，添加图片，视频，音频等。可编辑
     - 小记笔记： 用于记录一些碎片化的知识，例如会议记录，项目进度，个人学习记录等。可编辑，轻量化，本质是纯笔记，但是设置为小计笔记，可以方便后端使用ai进行分析和总结
     - 数据表笔记： 用于存储一些结构化数据，例如表格，列表等。可编辑，轻量化，本质是纯笔记，但是设置为数据表笔记，可以方便后端使用ai进行分析和总结
     - 后期会支持一些特定的模板，如记录收支等的模板
   - 知识库下面可以直接包含多个笔记或者是一个文件夹，文件夹下面可以包含多个文件夹或者笔记，最多支持五层。文件夹类型分为资源存储型文件夹以及普通文件夹
     - 资源存储型文件夹：用于存储手机资源存储型的知识库，例如图片，视频，音频等。不可编辑
     - 普通文件夹：用于存储普通类型的知识库，例如笔记，小记笔记，数据表笔记等，也可存储资源。可编辑
   - 可以通过拖动来编辑知识库组织结构
   - 知识库或者文件夹或者笔记创建时，用户可以自定义名称，类型，描述，并打上自己独有tag，方便后续检索
   - **\[新增] 版本管理与回收站**：笔记支持历史版本（快照 / diff 对比 / 一键回滚），删除进入回收站可恢复，避免误删。
   - **\[新增] 双向链接与引用**：支持 `[[笔记名]]` 式双向链接、块级引用（block reference），自动生成反向链接列表，为知识图谱提供人工关系补充。
   - **\[新增] 智能标签**：除用户手动 tag 外，AI 自动抽取关键词/实体作为候选标签，支持标签体系（层级标签、标签别名合并）。
   - **\[新增] 收藏与置顶、最近访问**：快速回到高频知识。
2. 文档解析与导出
   - 对于手机资源存储型的知识库：支持图片向量化解析，视频使用ai进行解析，音频使用ai进行解析
   - 对于所有类型笔记使用GraphRAG进行解析
   - 对于网站内容使用GraphRAG进行解析
   - 可以将用户上传的文档解析为知识库中的文档
   - 可以将知识库中的文档导出为不同的文件类型，例如pdf，ppt，excel文件等
   - 可以将整个知识库以zip文件的形式导出，也可以支持将整个知识库导出为一个pdf格式的文件，包含目录结构
   - 可以支持将整个扫描的pdf或者zip文件包转为知识库
   - 支持批量照片上传或者拍摄，批量转知识库或者笔记或者文件夹
   - 支持提供URL链接，解析为知识库中的文档
   - **\[新增] 统一解析管线**：文件上传 → 类型识别 → 文本抽取（OCR/ASR/多模态） → 分块（chunking） → 向量化 + 实体/关系抽取 → 入向量库/图库，全流程异步化（消息队列 + 任务状态可查）。
   - **\[新增] OCR 与版面解析**：扫描件/图片走 OCR（PaddleOCR），复杂 PDF 走版面分析（保留标题/表格/公式结构），提升 GraphRAG 抽取质量。
   - **\[新增] 音视频转写**：音频/视频先做 ASR（Whisper）转文字带时间戳，再做摘要与实体抽取；播放时可跳转到对应时间点。
   - **\[新增] 增量解析**：笔记编辑后仅对变更块重新向量化与图谱更新，控制成本。
   - **\[新增] 导出模板化**：PDF/PPT 导出支持主题模板（封面、目录、页眉页脚），Word/Markdown 双向转换。
3. 文档编辑
   - 支持markdown格式的文档编辑，并有类似语雀的编辑功能
   - **\[新增] 块编辑器**：基于块（Block）的富文本编辑（类 Notion），支持 `/` 命令快速插入代码块、公式（KaTeX/LaTeX）、表格、绘图（draw\.io 嵌入）、思维导图嵌入、可执行代码片段（代码沙箱）。
   - **\[新增] AI 行内辅助**：选中文本可触发 AI 快捷操作：续写、精简、翻译、转换格式（段落→要点列表/表格/时间线）、解释术语等。
   - **\[新增] 多人协同**（后期）：基于 CRDT（如 Yjs）的多端实时协同编辑，适用于未来社区/分享场景。
   - **\[新增] 离线编辑**：移动端支持离线编辑，网络恢复后增量同步，冲突检测与合并。
4. 记忆卡
   - 将笔记或者知识库内的文件夹下面的笔记批量转换为记忆卡类型
   - 记忆卡使用记忆曲线，提醒用户进行记忆，也可以自定义设置提醒时间，例如每天提醒，每周提醒
   - 可以设置特定文件夹或者笔记持续更新持续生成新的记忆卡
   - 记忆卡可以支持隐藏与显现内容，支持进入到特定的文档中的特定位置
   - 记忆卡可以设置提醒次数，例如提醒用户一次，提醒用户两次等
   - 记忆卡可以标记状态：已提醒，未提醒，已学习，未学习等，也可标记用户自定义的掌握程度，方便用户查看和管理
   - **\[新增] 间隔重复（SM-2/FSRS 算法）**：根据用户回答「熟悉/模糊/不熟悉」动态调整下次复习间隔，优先于固定时间提醒，提升记忆效率。
   - **\[新增] AI 自动生成记忆卡**：针对笔记内容自动抽取填空题、简答题、是非题形式的记忆卡，减少手工整理成本。
   - **\[新增] 记忆统计看板**：展示记忆进度热力图（类 GitHub contribution）、掌握率趋势、待复习数量统计。
5. AI小助
   - 支持用户输入问题，系统会根据用户库中的文档进行回答
   - 可以提供原文档材料出处证明回答的准确性
   - 可以调用一些MCP工具，比如设置定时提醒，发送短信提醒，发送邮件提醒等
   - 可以根据用户的会话历史，来推荐相关文档，或者推荐相关问题
   - 可以根据用户的自然语言，来为用户制定并设置一个计划：比如用户想养生：他自己有一个养生的笔记，然后他可以引用这个笔记，根据他的要求设定一个计划，比如每天提醒用户进行养生活动，或者每周提醒用户进行养生活动等。或者用户可以引用某个知识库或者某个笔记，来制定计划。
   - 学习模式：用户可以设置学习模式，比如学习模式下，系统会根据用户的会话历史，来推荐相关文档，或者推荐相关问题。用户可以在学习模式下，来学习新的知识，也可以在学习模式下，来复习已经学习的知识。
   - **\[新增] 混合检索（Hybrid Search）**：回答问题时同时走 稠密向量检索（向量库）+ 稀疏关键词检索（ES/BM25） + 图谱多跳推理（GraphRAG），结果融合后作为上下文，兼顾精确匹配与语义推理。
   - **\[新增] 多文档综合分析**：支持用户指定多个笔记/知识库作为上下文，AI 跨文档归纳、对比、生成综述报告（输出为新笔记）。
   - **\[新增] 对话式知识入库**：用户和 AI 对话的过程中，可以一键将某段回答或总结另存为笔记，对话历史本身也可索引为知识。
   - **\[新增] 主动推送（Daily Brief）**：每日定时分析用户今日待复习记忆卡 + 相关知识更新 + 计划提醒，以通知/摘要卡片形式主动推送给用户。
   - **\[新增] 知识盲区发现**：AI 基于知识图谱分析用户当前知识结构的薄弱连接，主动推荐「你可能还不了解的关联知识」。
6. Agent配置
   - 支持类似Dify的图编辑模式来配置Agent的工作流
   - 每个节点可以自定义功能：可以选择执行时间，可以自定义Skill，或者选择已有的Skill进行调用，可以选择我服务器提供的MCP能力进行调用，也可配置自己的MCP能力进行调用，用来分析文档，生成数据，或者执行其他任务
   - 可以在节点之间添加条件分支，根据节点的输出结果，来判断是否执行下一个节点
   - 可以在节点之间添加循环，根据节点的输出结果，来判断是否重复执行下一个节点
   - 也可以再每个节点通过自然语言输入来让ai解析出该节点的工作流配置
   - **\[新增] 工作流模板市场**：内置常用工作流模板（每日总结、新文档入库通知、定期知识报告生成、学习计划跟踪等），用户可一键安装使用或基于模板修改。
   - **\[新增] 事件触发**：除定时调度外，支持事件触发（笔记新增/更新、标签命中、记忆卡到期等）作为工作流入口，形成知识-行动闭环。
   - **\[新增] 工作流调试与日志**：节点执行历史、输入输出日志、错误重试，方便用户排查工作流问题。

## 知识图谱可视化

> 这是 GraphRAG 的核心价值呈现，强烈建议作为核心特性开发。

- **图谱总览视图**：以力导向图（force-directed graph）展示全库实体关系网络，节点按实体类型着色（人物/技术/概念/项目等），边标注关系类型。
- **局部展开**：从任意笔记或实体出发，展示其一到多跳邻居，快速发现关联知识。
- **时间轴视图**：知识入库时间线视图，直观看到知识积累过程。
- **聚类视图**：AI 自动按主题聚类，识别知识孤岛（没有关联的孤立笔记），提示用户补充关联。
- **路径查询**：输入两个概念，图谱找出它们之间的关联路径并解释，例如「Java虚拟机」→「GC算法」→「性能调优」。

## 智能检索

- **全文检索**：基于 Elasticsearch 的全文搜索，支持高亮、模糊匹配、中文分词（ik\_analyzer）。
- **语义检索**：基于向量相似度检索，用自然语言描述找相关内容。
- **图谱检索**：通过实体名/关系类型在知识图谱中检索，支持多跳查询。
- **混合检索融合**：RRF（Reciprocal Rank Fusion）将以上三路结果重排序，综合提供最相关结果。
- **筛选与排序**：按时间、标签、知识库、文件类型、AI 置信度等维度过滤，支持保存常用筛选条件。
- **搜索历史与推荐**：记录搜索行为，推荐相关问题；同一搜索词下可看到「你之前查过的相关笔记」。

## 统计与个人洞察

- **知识仪表盘**：总笔记数、知识库数、图谱节点数、本周新增数、记忆卡掌握率等核心指标。
- **学习热力图**：类 GitHub contribution 图，展示每天知识录入与复习活跃度。
- **知识结构分析**：各主题占比（饼图/树图），标签词云，知识增长趋势折线图。
- **AI 周报**：每周自动生成「本周知识摘要」，总结新增知识点、完成复习情况、推荐下周学习方向。

## 分享与社区（后期规划）

- **笔记/知识库公开分享**：生成只读分享链接或二维码，支持设置有效期和访问密码。
- **知识名片**：用户个人主页展示公开知识库和知识图谱摘要，作为个人知识品牌展示。
- **知识订阅**：关注其他用户的公开知识库，有更新时收到通知。
- **协作知识库**（后期）：多用户共同维护一个知识库，类似 Notion 团队空间。

# 非功能需求

## 安全性

- 包括一些传统应用的特点：加密上传，鉴权登录，用户注册，用户管理等
- **传输加密**：全站 HTTPS / TLS 1.3
- **存储加密**：敏感文件服务端加密（SSE），可选客户端加密（端到端，服务器无法读取明文）
- **数据隔离**：多租户严格隔离，向量库/图库按 user\_id namespace 隔离
- **API 安全**：Rate Limiting、输入校验、SQL/Prompt Injection 防护
- **隐私合规**：支持账号注销+数据彻底删除（GDPR / 国内个保法合规）

## 性能与可用性

- 文档解析异步化，上传即返回任务 ID，用户可查看解析进度
- AI 对话流式输出（SSE），首 Token 延迟 < 2s
- 向量检索 P99 < 200ms，全文检索 P99 < 100ms
- 移动端核心页面离线可用

## 可观测性

- 全链路 Trace（OpenTelemetry）
- LLM 调用成本统计与预警（Token 消耗监控）
- 用户行为埋点（知识录入/检索/复习频率分析）
